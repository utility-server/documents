## üåü Overview

The **TTS Utility Server API** is a robust, enterprise-grade RESTful API designed to provide comprehensive solutions for:

- **Authentication & Authorization**: Secure user login and session management
- **Key Management**: Dynamic access key and secret key rotation
- **Real-time Monitoring**: Live logs and hit tracking
- **Analytics**: Detailed usage statistics and reports
- **Cache Management**: Advanced caching operations via Spider Module
- **Content Delivery**: Sitemap management and TTL configuration

Built with performance, security, and scalability in mind, this API serves as the backbone for high-traffic applications requiring reliable authentication and content delivery mechanisms.

## üöÄ Getting Started

### Prerequisites

Before using the API, ensure you have:

1. **Client Identifier (`identifire`)**: Your unique client ID
2. **Access Key**: API access key for authentication
3. **Secret Key** (optional): Additional security key for sensitive operations
4. **API Client**: Any HTTP client (curl, Postman, or programming language HTTP library)

### Base URLs

Choose the appropriate base URL for your environment:

**Production:**
```
https://prod.services.utility-server.com/{your-identifire}
```

**Development:**
```
http://btv.localhost:3000/{your-identifire}
```

**Spider Module:**
```
https://spider.services.utility-server.com
```

### Installation

No installation required! This is a RESTful API accessible via HTTP requests.

For integration into your application, install your preferred HTTP client:

**Node.js:**
```bash
npm install axios
# or
yarn add axios
```

**Python:**
```bash
pip install requests
```

**PHP:**
```bash
composer require guzzlehttp/guzzle
```

```bash
curl -X POST "https://prod.services.utility-server.com/www.example.com/customers/member_login" \
  -H "Content-Type: application/json" \
```

```javascript
const axios = require('axios');
const crypto = require('crypto');

// Generate checksum
function generateChecksum(data) {
  const jsonString = JSON.stringify(data);
  return crypto.createHash('sha256').update(jsonString).digest('hex');
}

// Login function
async function login(email, password, identifire, secret, accessKey) {
  const data = {
    email_id: email,
    password: password,

    client_ver: 'V4.21.04',
```

<summary><b>Python</b></summary>

```python
import requests
import hashlib
import json

def generate_checksum(data):
    """Generate SHA-256 checksum for data integrity"""
    json_string = json.dumps(data, sort_keys=True)
    return hashlib.sha256(json_string.encode()).hexdigest()

def login(email, password, identifire, secret, access_key):
    """Login to TTS Utility Server"""
    url = f"https://prod.services.utility-server.com/{identifire}/customers/member_login"
    
    data = {
        "email_id": email,
        "password": password,
        "identifire": identifire,
        "client_ver": "V4.21.04",
        "ip_address": "192.168.1.1",
        "mac_address": "-",
        "secret": secret
    }
    
    data["checksum"] = generate_checksum(data)
    
    headers = {
        "Content-Type": "application/json",
        "User-Agent": "TTS-Utility-Client/V4.21.04",
        "x-tts-cc-access-key": access_key
    }
    
    try:
        response = requests.post(url, json=data, headers=headers)
        response.raise_for_status()
        print("Login successful:", response.json())
        return response.json()
    except requests.exceptions.RequestException as e:
        print(f"Login failed: {e}")
        raise

# Usage
login('user@example.com', 'password123', 'www.example.com', 'SECRET_KEY', 'ACCESS_KEY')
```

```php
<?php
require 'vendor/autoload.php';

use GuzzleHttp\Client;

function generateChecksum($data) {
    $jsonString = json_encode($data, JSON_UNESCAPED_SLASHES);
    return hash('sha256', $jsonString);
}

function login($email, $password, $identifire, $secret, $accessKey) {
    $client = new Client();
    
    $data = [
        'email_id' => $email,
        'password' => $password,
        'identifire' => $identifire,
        'client_ver' => 'V4.21.04',
        'ip_address' => '192.168.1.1',
        'mac_address' => '-',
        'secret' => $secret
    ];
    
    $data['checksum'] = generateChecksum($data);
    
    try {
        $response = $client->post(
            "https://prod.services.utility-server.com/{$identifire}/customers/member_login",
            [
                'json' => $data,
                'headers' => [
                    'Content-Type' => 'application/json',
                    'User-Agent' => 'TTS-Utility-Client/V4.21.04',
                    'x-tts-cc-access-key' => $accessKey
                ]
            ]
        );
        
        $result = json_decode($response->getBody(), true);
        echo "Login successful: " . json_encode($result) . "\n";
        return $result;
    } catch (Exception $e) {
        echo "Login failed: " . $e->getMessage() . "\n";
        throw $e;
    }
}

// Usage
login('user@example.com', 'password123', 'www.example.com', 'SECRET_KEY', 'ACCESS_KEY');
?>
```

<details>
<summary><b>How do I get API credentials?</b></summary>

Contact our sales team at sales@utility-server.com to request API credentials. You'll receive:
- Client identifier (identifire)
- Access key
- Secret key
- Onboarding documentation
</details>

<details>
<summary><b>Is there a free tier available?</b></summary>

Please contact sales@utility-server.com for information about pricing tiers and free trial options.
</details>

<details>
<summary><b>What's the difference between access key and secret key?</b></summary>

- **Access Key**: Used for general API authentication, included in request headers
- **Secret Key**: Additional security layer for sensitive operations like login and key rotation
</details>

<details>
<summary><b>How long do sessions last?</b></summary>

Sessions have an inactive timeout of 43,200 minutes (30 days) by default. The timeout resets with each API call using the session.
</details>

<details>
<summary><b>Can I use the API in production?</b></summary>

Yes! The API is production-ready and currently serves high-traffic applications. Ensure you follow our [Best Practices](./API_DOCUMENTATION.md#best-practices) guide.
</details>

<details>
<summary><b>How do I generate checksums?</b></summary>

Checksums are SHA-256 hashes of your JSON request body. See examples in the [Quick Example](#quick-example) section.
</details>

<details>
<summary><b>What should I do if I get rate limited?</b></summary>

Wait for the time specified in the `Retry-After` header, then retry your request. If you consistently hit rate limits, contact support@utility-server.com to discuss increasing your limits.
</details>
